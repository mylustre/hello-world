18 注：从远程分支 checkout 出来的本地分支，称为跟踪分支(tracking branch)。跟踪分支是一种和远程分支有直接联系的本地分支。在跟踪分支里输入git push，Git 会自行推断应该向哪个服务器的哪个分支推送数据。反过来，在这些分支里运行git pull 会获取所有远程索引，并把它们的数据都合并到本地分支中来。git config -l 可以看到本地分支和远程分支的跟踪关系   
在克隆仓库时，Git 通常会自动创建一个名为 master 的分支来跟踪 origin/master。这正是git push 和 git pull 一开始就能正常工作的原因。当然，你可以随心所欲地设定为其它跟踪分支，比如origin 上除了 master 之外的其它分支。

git 使用场景及操作（  将本地代码上传到服务器的一个新项目中，新项目除了master分支，还新建了一个newfeature分支）
1.在git服务器找到项目，复制clone地址（https的）
2.本地目录新建目录，newFold，进入目录后启动git bash here   
3.gitclone    http://XXX@github.com:XXX/XXX.git  
4.新建并切换分支：git checkout -b newfeature  (git branch newFeature ,git check newFeature)
5.git pull  (提示  需要建立关联跟踪   git branch --set-upstream-to=origin/newFeature newFeature）
6.再次git pull  （显示Already up to date）  一切就绪，可以在当前目录新建文件，并push了，把需要上传的代码放入当前目录。
7.git add .
8.git commit -m ' add ...'
9.git push


以上4,5,6可以简化为一步
git checkout -b newFeature origin/newFeature (新建分支，设定远程关联关系，pull代码到本地，并切换到新分支）
